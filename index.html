box-sizing: border-box

The purpose of border-box involves the box model, where the padding and border are included in the height of the element. With border-box, if you enter width: 100px; height:100px;the element will be exactly 100px by 100px. To showcase this, if you have padding:10px; border:10px solid; height:100px; width:100px .Without border box the total width and height will be 140px x 140px since we gave the content 100px x 100px but then the padding and border are added after, changing the height and width. With border box, the height x width of the content will be 60px x 60px; with the padding and border taking up their 20px x 20px each.The height and width of the element of the whole will now be 100px x 100px as intended

To further explore this, this is taken from Web Docs. "In the standard box model, if you give a box a width and a height attribute, this defines the width and height of the content box. Any padding and border is then added to that width and height to get the total size taken up by the box. The actual space taken up by the box will be 410px wide (350 + 25 + 25 + 5 + 5) and 210px high (150 + 25 + 25 + 5 + 5)."

.box { width: 350px; height: 150px; margin: 10px; padding: 25px; border: 5px solid black;}

Justify-content

Justify-content works with placing flex items without having to change the width on the main axis.

  • Space-between

    • Space between makes sure that each space between them are laid out evenly, any after that is not between another item is not accounted for, which is why you have the first item and last item pushed all the way to the ends.

  • Space-around

    • Space around causes the spacing between items to be the same, but gives the last and first item equal space to the left and right of the ends.

  • Space-evenly

    • Space-evenly causes all spaces to be even, including the end points.

Common Grid Container Properties

  • grid-template-columns or rows - gives the spacing of the grid tracks, similar to padding. Can be given names with brackets.
  • grid-template-columns repeat - used if there are repeating parts. How many times do we want to call it. Ie, we want 3 items to each taken a fraction of the space.
  • grid-template-areas - when we define the items in the container grid with grid-area, we can give the main grid the property of grid-template area to do very complex of alignment of the items.Note, we do have to name every item to allow this to work, and we need each row needs the same number of cells/items per line if we don't add the dot notation, which means nothing.
  • grid-gap - to create gaps, we will use column-gap or row-gap. Grid-column/row-gap will no longer be supported. For a shorthand, you can just use the word gap.
  • Align items left to right - with justify items, align items top to bottom with align items. Align self works the same but with the effect of only one items. Place items is the shorthand for both in order of align then justify, just think top first then left and right.
  • In the event that your items don't fill your full container, like what happens when we use px, we will use justify-content and align content. For shorthand, use place-content.
  • Grid-auto-columns/rows - are used when we place an item that's not in our created grid layout. This essentially tell the grid to create these empty spaces with so it can fit in the right spot.

Common Grid Items properties

  • grid-column-start/end and grid-row-start/end - these can be a number, where they will start and end given the line, or you can use span and then a number and it will go expand over those grid tracks.
  • justify-self and align self - similar to justify content and align items but with only one item. Place-self sets both.
  • min-max - allows rows and columns to only shrink so much. When using template repeat, we should be setting a min-max to avoid grid blowout.

What is Intersection Observer

Intersection Observer is a powerful tool that allows on scroll effects without the need for it to be called each time a scroll is done.

How does Intersection work?

Most times, intersection observer can be used with query selector to select all elements or class names, or single elements and a class name. Just note that if you do select multiple elements and classes, the steps are slightly different.

options hold three values; root, threshold, and margin

  • root - can be left out since most times root will be set to null, or just not written out at all. Both not writing anything for null and setting it to null will both set it to the viewport as seen in the skeleton example.
  • threshold - a 0 to 1 scale. By default 0. 0 is when any piece of the section is in view, while 1 is when 100% of the section is visible.
  • rootMargin - works like margins in CSS. If wanted, it must be in parenthesis and must be in either percentage or px base. Dictates when it's fired.

When to use declaration vs function expression?

When to choose Function Declaration versus Function Expression? As a rule of thumb, when we need to declare a function, the first thing to consider is Function Declaration syntax. It gives more freedom in how to organize our code, because we can call such functions before they are declared. That's also better for readability, as it's easier to look up function f(…) {…} in the code than let f = function(…) {…};. Function Declarations are more “eye-catching”. …But if a Function Declaration does not suit us for some reason, or we need a conditional declaration. Another note, function expressions can only work if placed first, while declarations can work any time, but only within the scope of the function if it was made in one.

  2. Plan:What inputs does the program have?Where is the data? What's the desired output? And given your inputs, what are the steps to get the desired output? This last question is where the code begins.Great place to use comments in JS code.
  3. Pseudocode: Write out the plan of attack in plain english first. An example of this will be if we are trying to print out inputted numbers.

    "When the user inputs a number Initialize a counter variable and set its value to zero. While counter is smaller than user inputted number increment the counter by one. Print the value of the counter variable."

  4. Decomposition: Decomposition is the main way to deal with complexity.Work on the subproblem first, trying to make one function for the whole thing makes issues and knots later one.

FizzBuzz

"Write a program that takes a user's input and prints the numbers from one to the number the user entered. However, for multiples of three print Fizz instead of the number and for the multiples of five print Buzz. For numbers which are multiples of both three and five print FizzBuzz."

  1. Understand: Write a program that allows the user to enter a number, print each number between one and the number the user entered, but for numbers that divide by three without a remainder print Fizz instead. For numbers that divide by 5 without a remainder print Buzz and finally for numbers that divide by both three and five without a remainder print FizzBuzz.
  2. Plan: Does your program have an interface? What will it look like? Our FizzBuzz solution will be a browser console program, so we don't need an interface. The only user interaction will be allowing users to enter a number. What inputs will your program have? Will the user enter data or will you get input from somewhere else? The user will enter a number from a prompt (popup box). What's the desired output? The desired output is a list of numbers from 1 to the number the user entered. But each number that is divisible by 3 will output Fizz, each number that is divisible by 5 will output Buzz and each number that is divisible by both 3 and 5 will output FizzBuzz. Given your inputs, what are the steps necessary to return the desired output?
  3. Pseudocode: When a user inputs a number Loop from 1 to the entered number If the current number is divisible by 3 then print "Fizz" If the current number is divisible by 5 then print "Buzz" If the current number is divisible by 3 and 5 then print "FizzBuzz" Otherwise print the current number
  4. Decomposition:As we can see from the algorithm we developed, the first subproblem we can solve is getting input from the user. So let's start there and verify it works by printing the entered number. Next loop from 1 to the enter number. Next if it can be divided by 3 then print fizz. etc. For the rest of the explanation go to The Odin Project
